# Reference: Design Theory

Longer-form example content used to demonstrate how a reference document reads
in the detail drawer. None of this describes a real system.

## Lead with the action

A good guide tells the reader what to do before explaining why. Open each
section with an imperative sentence, then add context only if it changes the
action.

## One idea per section

Sections should be skimmable. If a section needs more than one callout, it is
probably two sections.

## Examples over prose

A short, runnable example communicates faster than a paragraph. Pair every
non-obvious option with an example of it in use.

## Keep references separate

Deep material — option tables, edge cases, rationale — belongs in
`references/` so the main `SKILL.md` stays short and actionable.
